# RoEzConfig - LarkExcel2Studio

> 将飞书电子表格一键导入 Roblox Studio 的开源工具

---

## 适用范围

- 使用飞书云文档的 Excel 作为配置表，方便多人协作
- 不使用 Rojo + VSCode，直接在引擎内编程
- 启用多人协作和草稿模式（非必需，但建议）

---

## 功能介绍

- 从飞书电子表格拉取数据，自动转换为 Lua ModuleScript
- 支持多张配置表批量导入
- 支持多个 Sheet 的表格
- 本地工具 + Studio 插件协同工作，稳定可靠
- 配置方案可导出分享给团队成员

---

## 快速开始

### 第一步：安装 Studio 插件

将 `极导表插件文件夹v1.0.0` 文件放入以下目录：
C:\Users\你的用户名\AppData\Local\Roblox\Plugins\

重启 Roblox Studio，顶部工具栏会出现「极导表」按钮。

### 第二步：启动本地工具

双击运行 `极导表_本地协同插件v1.0.0`，浏览器会自动打开管理界面，系统托盘会出现图标。

### 第三步：配置飞书凭据

1. 在 [飞书开放平台](https://open.feishu.cn) 创建一个企业自建应用
2. 开启以下权限：
   - `sheets:spreadsheet`（电子表格读取）
3. 在管理界面填入 **App ID** 和 **App Secret**
4. 点击**测试连接**，显示「连接成功」即可

### 第四步：添加配置表

点击**+ 添加**，填写以下信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| 名称 | 配置表的备注名称 | 怪物表 |
| 飞书 Token | 电子表格 URL 中的 Token | `shtcnxxxxxxxx` |
| 目标路径 | Studio 工程中的存放路径 | `ReplicatedStorage.Data` |
| 文件名 | 生成的 ModuleScript 名称 | `MonsterConfig` |

### 第五步：导入

- 点击单张表的**导入**按钮，导入单张表
- 点击**全部导入**，一次性导入所有配置表
- Studio 插件面板中打开插件，会监听并显示写入结果

## 表格格式约定

飞书电子表格需要按照以下格式组织数据：

| 行 | 内容 |
|---|---|
| 第 1 行 | 字段名（英文） |
| 第 2 行 | 中文备注（导入时跳过） |
| 第 3 行 | 字段类型（`number` 或 `string`） |
| 第 4 行起 | 数据行，第 1 列为主键 |

> 列名为 `Note` 的列会自动跳过，不导入。

## 团队协作

配置好所有表格信息后，点击**导出方案**，会在工具同目录下生成 `ExcelConfig.json`。

将此文件分享给团队成员，成员点击**加载方案**即可一键同步所有配置。

## 端口说明

| 端口 | 用途 |
|---|---|
| 默认 11451 | Studio 插件轮询端口 |
| 默认 11452 | 网页管理界面端口 |

如需修改端口，在网页端和 Studio 插件端分别保存新端口，并重启程序后生效。

## 常见问题

**Q：点击导入后 Studio 没有反应？**
A：检查 Studio 插件面板是否已打开，以及插件端口是否和网页端一致。

**Q：连接飞书失败？**
A：检查 App ID 和 App Secret 是否正确，以及飞书应用是否已发布并开启了表格读取权限。

**Q：导入的数字变成了科学计数法？**
A：确认表格第 3 行的类型填写的是 `number`，工具会自动处理整数和小数格式。
